# 星穹铁道行动值与速度系统设计文档

## 1. 核心数据定义

### 1.1 战斗单位 `Unit`

仅供参考，以现有系统为准

```python
class Unit:
    name: str
    base_speed: float          # 基础速度
    speed_bonus: float         # 速度加成（百分比，如0.12表示12%）
    speed_fix: float           # 固定速度加成（如25）
    is_player: bool            # 是否为我方角色
    action_value: float        # 当前行动值（AV），初始通过计算获得
    
    # 其他战斗属性（攻击、血量等）略
```

### 1.2 速度计算公式

```
最终速度 = base_speed × (1 + speed_bonus) + speed_fix
```

当加成变化时（如队友提供速度buff），**立即重新计算最终速度**，并根据后述规则更新该单位当前的 `action_value`。

---

## 2. 行动值初始化

### 2.1 跑道总长

定义常量：  

```
TRACK_LENGTH = 10000
```

所有单位在一条长度为 `10000` 的跑道上奔跑。

### 2.2 初始行动值计算

每个单位进入战斗时，计算其首次行动所需时间：

```
initial_av = TRACK_LENGTH / 最终速度
```

系统将所有单位的 `action_value` 设置为该值，并排序。

**示例：**

- 希儿 速度115 → AV = 10000 / 115 ≈ 86.9565
- 主角 速度100 → AV = 100
- 布洛妮娅 速度99 → AV ≈ 101.0101

排序后 AV 最小的单位最先行动。

---

## 3. 行动轮转逻辑

### 3.1 选出当前行动者

从所有存活单位中，选择 `action_value` 最小的单位。若多个单位 AV 相同，按以下优先级决定：

1. 我方角色 > 敌方
2. 同阵营时，站位索引小的优先（见后述站位说明）

确定当前行动者后，进入**行动阶段**。

### 3.2 行动消耗时间

当前行动者执行基础回合（可发动普攻、战技等）所消耗的时间等于**其当前的 action_value**。记该值为 `time_cost`。

### 3.3 全场 AV 更新

行动结算时，执行以下步骤：

**步骤1：当前行动者重置**

该单位完成一圈，回到起点，计算新一圈的 AV：

```
current.action_value = TRACK_LENGTH / current.最终速度
```

**步骤2：全体其他单位推进**

所有其他单位的 AV 减去 `time_cost`：

```
unit.action_value = unit.action_value - time_cost
```

结果可以小于0，此时表示该单位“已经冲过了终点”，将在下一轮立即行动（或插入多动）。

**步骤3：重新排序**

将全场单位按 `action_value` 从小到大重新排序。

### 3.3.1 循环示例

初始：

- 希儿 86.9565
- 主角 100
- 布洛妮娅 101.0101

第一动：希儿行动，time_cost = 86.9565

- 希儿新 AV = 86.9565（速度未变）
- 主角 = 100 - 86.9565 = 13.0435
- 布洛妮娅 = 101.0101 - 86.9565 ≈ 14.0536

排序后：主角 13.0435 → 布洛妮娅 14.0536 → 希儿 86.9565

第二动：主角行动，time_cost = 13.0435

- 主角新 AV = 100
- 布洛妮娅 = 14.0536 - 13.0435 ≈ 1.0101
- 希儿 = 86.9565 - 13.0435 = 73.9130

排序后：布洛妮娅 1.0101 → 希儿 73.9130 → 主角 100

以此循环。

## 4. 拉条机制（Action Advance）

### 4.1 定义

拉条效果以百分比形式出现，表示强制推进目标在跑道上的绝对距离。  
百分比基于 `TRACK_LENGTH` 计算。

```
advance_distance = 拉条比例 × TRACK_LENGTH
```

例如：40% 拉条 = 0.4 × 10000 = 4000 距离。

### 4.2 实现方式

对目标单位施加拉条时，立即修改其当前 `action_value`：

```
target.action_value = target.action_value - advance_distance
```

如果结果 ≤ 0，该单位会立即插入行动队列，**在当前行动者完成动作后，以“额外回合”的形式行动**，而不会产生额外的 `time_cost`。

### 4.3 特殊情形：100%拉条（立即行动）

立即行动可视为 advance_distance = 10000 的拉条。  
直接设置 `target.action_value = -1`（或一个极小的负数），确保其成为下一个行动者。

**额外回合规则：**

- 额外回合同样以 `TRACK_LENGTH / 速度` 重置 AV。
- 但该额外回合**不会消耗时间**（即不会导致全体减 AV）。
- 额外回合结束后，从被中断的起点继续恢复正常的 AV 排序。

### 4.4 连续拉条处理

如果多个单位在同一时间点被拉条至 AV ≤ 0，它们的行动顺序按照拉条生效的先后顺序依次排入额外回合队列。

## 5. 推条机制（Action Delay）

### 5.1 定义

推条效果以百分比形式表示，令目标向后退一定距离。

```
delay_distance = 推条比例 × TRACK_LENGTH
```

例如：30% 推条 = 0.3 × 10000 = 3000。

### 5.2 实现方式

```
target.action_value = target.action_value + delay_distance
```

### 5.3 速度减低的影响

如果推条效果伴随“减速”，先应用速度变化重算 AV，再应用推条。具体流程：

1. 计算推条前的 AV：`av_before = target.action_value`
2. 修改速度，更新最终速度 `new_spd`
3. 计算速度变化造成的时间延后量：  
   `speed_delay = av_before × ( 旧最终速度 / 新最终速度 - 1 )`
4. 应用推条距离：  
   `target.action_value = av_before + speed_delay + delay_distance`

此设计保证减速与推条的计算顺序与游戏内一致。

## 6. 终结技插队

### 6.1 规则

终结技可在以下任意时刻释放（类似反应触发）：

- 我方角色行动时
- 敌方单位行动时
- 任意行动间隔的“空闲点”（上一动完毕而下一动开始前）

### 6.2 实现要点

- 终结技不消耗任何行动值，不更新全场 `time_cost`。
- 释放终结技时，所有单位的 `action_value` 保持不变。
- 终结技释放完毕后，战斗从之前暂停的时间点继续。若在空闲点释放，则继续选取 AV 最小的单位行动。

Agent 需实现一个“待机释放队列”，当玩家或AI命令触发终结技时，暂停正常的 AV 流转，执行完毕后恢复。

## 7. 轮次与行动次数阈值

### 7.1 轮次定义

在 “忘却之庭” 等限时模式中，存在固定时长的轮次限制：

- **第一轮**：时长 **150 AV**
- **第二轮及以后**：每轮时长 **100 AV**

即总可用 AV：

```
round_av_limit[1] = 150
round_av_limit[n] = 150 + (n-1) × 100   (n >= 2)
```

### 7.2 行动次数计算

若需要在特定轮次内让某单位达到目标行动次数，可通过速度或拉条调整。

**纯速度下的基准公式：**

给定速度 `v`，第 `n` 轮结束时经过的总 AV 为 `T_n`，则该单位完成的完整圈数为：

```
行动次数 = floor( T_n / (TRACK_LENGTH / v) ) = floor( T_n × v / 10000 )
```

对于首轮（150 AV）：

- 首轮 2 动要求：150 × v / 10000 ≥ 2 → v ≥ 133.33  
  因此面板速度 **≥ 134** 可实现首轮 2 动。
- 次轮（合计 250 AV）内达到 4 动需：250 × v / 10000 ≥ 4 → v ≥ 160  
  因此速度 **≥ 160** 可在 2 轮内 4 动。

**常见速度阈值表：**

| 目标动作次数 | 对应轮数 | 最低速度 | 说明 |
|--------------|----------|----------|------|
| 首轮2动      | 1        | 134      | 角色面板速度需≥134 |
| 4轮7动       | 4        | 160      | 总计行动7次 |
| 5轮8动       | 5        | 約156    | 参考值 |
| ...          | ...      | ...      | 可根据公式推算 |

### 7.3 拉条等效速度

拥有固定拉条效果（如风套、舞舞舞）时，可将其折算为等效速度，降低面板需求。实现时，Agent 应直接通过模拟或公式计算是否达到目标次数，而非硬编码阈值。

## 8. 配速与站位规则

### 8.1 站位索引

我方角色的站位（从左到右，1-4号位）在判定行动顺序时具有隐藏优先级：  
当多个单位 **AV 完全相同** 时，站位索引小的角色先行动。

敌方单位同样按照预设的站位顺序（或固定顺序）行动。

### 8.2 配速建议生成

速度梯度建议：辅助 > 副C > 主C，以保证增益先覆盖。  
Agent 可根据此规则输出配速建议，但战斗模拟核心只需严格按照 AV 排序 → 站位优先级的规则执行。

## 9. 全局状态流转伪代码

```python
def battle_loop(units):
    # 初始化
    for u in units:
        u.action_value = TRACK_LENGTH / get_final_speed(u)
    sort_by_av(units)
    
    while battle_not_over:
        current = units[0]  # AV最小者
        
        # 等待玩家/ai指令（可选插入终结技）
        interrupt_by_ult = check_ult_interrupt()
        if interrupt_by_ult:
            execute_ult(interrupt_by_ult)  # action_value不修改
            sort_by_av(units)              # 若有自拉条/推条效果则重排
            continue
        
        # 正常行动
        time_cost = current.action_value
        perform_action(current)
        
        # 更新全场AV
        current.action_value = TRACK_LENGTH / get_final_speed(current)
        for u in units:
            if u != current:
                u.action_value -= time_cost
        sort_by_av(units)
        
        # 检查胜负
        if is_victory_or_defeat():
            break
```

## 10. 关键函数清单

| 函数 | 功能 |
|------|------|
| `get_final_speed(unit)` | 根据 base_speed, speed_bonus, speed_fix 计算最终速度 |
| `init_av(units)` | 为所有单位计算初始 action_value |
| `sort_by_av(units)` | 按 action_value 升序，相同则按站位优先级排序 |
| `apply_advance(target, ratio)` | 执行拉条，更新 target.action_value，必要时插入额外回合队列 |
| `apply_delay(target, ratio, speed_debuff=0)` | 执行推条及可选减速，更新 AV |
| `handle_ult_interrupt()` | 终结技插队处理，不消耗时间 |
| `update_av_after_action(current, units)` | 行动后重置行动者AV，并全场减去 time_cost |
| `get_actions_in_cycle(unit, cycle_count)` | 计算给定轮次内的行动次数（用于配速验证） |
